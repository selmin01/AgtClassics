Quick Sort

O Quicksort é um algoritmo de ordenação interna mais rápido que se conhece para uma ampla variedade de situações, provavelmente é o mais utilizado. Dividir o problema de ordenar um conjunto com n itens em dois problemas menores no qual são ordenados independentemente, e as partições são combinadas para produzir uma solução final. A escolha do pivô é um dos grandes desafios ao implementar o Quicksort. Podemos escolher o elemento inicial, o elemento final ou até mesmo a mediana de três valores como pivô. As possibilidades são numerosas. Uma delas consiste em escolher um elemento aleatório do vetor como pivô

  • A complexidade de tempo do pior caso da ordenação é O(N^2)
  • A complexidade de tempo de caso médio é O(n log(n))
  • A complexidade de tempo do melhor caso é O(n log(n))
  • A complexidade do espaço é O(n)

  • Vantagens:
      • Melhor opção para ordenar vetores grandes
      Muito rápido por que o laço interno é simples
      Memória auxiliar para a pilha de recursão é pequena
      Complexidade no caso médio é O(n lg(n))
  • Desvantagens:
      • Não é estável (não conhecemos forma eficiente para tornar o quicksort estável)
      Pior caso é quadrático
    O pior caso acontece quando o pivô é sempre o maior ou menor elemento (partições de tamanho desequilibrado)
    Melhor caso acontece quando as partições têm sempre o mesmo tamanho (partições balanceadas) O(n log(n)).
